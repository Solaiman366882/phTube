Var, Let, এবং Const এর পার্থক্য?

ES2015 (ES6) এর সাথে অনেক গুলো নতুন ফিচার এসেছে । ES6 এর সাথে আসা ফিচারগুলোর একটি হল let এবং const কীওয়ার্ড, যেগুলোকে ভ্যারিয়েবল ডিক্লেয়ার করার সময় ব্যবহার করা যায়।
var: var ভ্যারিয়েবলগুলো রিডিক্লেয়ার এবং আপডেট করা যায় এবং var এর কিছু সমস্যা আছে,এর কারণে আপনার কোডে অনেক বাগও তৈরি হতে পারে। এজন্যই let এবং const ব্যবহার করা জরুরি।
let: ভ্যারিয়েবল ডেক্লারেশনের জন্য এখন let বেশি ব্যবহার করা হয়। let হল block scoped let এবং আপডেট করা যায় তবে রিডিক্লেয়ার করা যায় না ।
const: const ব্যবহার করে ডিক্লেয়ার করা ভ্যারিয়েবলগুলো হল ধ্রুবক অর্থাৎ অপরিবর্তনীয়। const ডেক্লারশনগুলো অনেকাংশে let এর মতই। const দিয়ে ডিক্লেয়ার করা ভ্যারিয়েবল অপরিবর্তিত থাকবে। এটা আপডেট অথবা রিডিক্লেয়ার করা যাবে না।

API জিনিসটা কী ?

Rest API - Representational State Transfer Application Programming Interface
API: সহজ করে বলতে গেলে যে পদ্ধতিতে অন্য কোন সফটওয়্যার বা প্লাটফ্রম থেকে ডাটাকে বা ফিচার নিজেদের সফটওয়্যার , প্লাটফ্রম এনে ব্যবহার করতে পাড়ি , যে সিস্টেমে ডাটা বা ফিচার নিজের সফটওয়্যার বা ওয়েব এপ্লিকেশন নিয়ে আসা হয় সেটাই API
REST: REST মানে হল Representational State Transfer, অর্থাৎ প্রত্যেকটি URL কিছু অবজেক্টকে রিপ্রেজেন্ট করে । রেস্ট এর সবচেয়ে ছোট সঙ্গা এর নামেই দেয়া আছে, “Representational State Transfer” অর্থাৎ এটি কোন কিছুর অবস্থা বা স্টেট একটি রিপ্রেজেন্টেশন এর মাধ্যমে ক্লায়েন্ট টু সার্ভার এবং সার্ভার টু ক্লায়েন্টে ট্রান্সফার করে ।

null এবং undefined

undefined: যখন একটা ভেরিয়েবল ডিক্লেয়ার করা হয় কিন্তু এতে কিছু assign করা হয়না বা initialize করা না হয় তখন এই ভেরিয়েবলটির ডেটা টাইপ "undefined" হবে। এই ডেটা টাইপের একটাই মান undefined
null: null আর undefined প্রায় একই, একটা পার্থক্য হচ্ছে কোন ভেরিয়েবলের মান null assign করতে পারে কিন্তু undefined দিতে পারবেন না। এই ডেটা টাইপের মান একটাই অর্থ্যাৎ null.
